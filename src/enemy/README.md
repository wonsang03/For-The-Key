# 🔑 For the Key: 2D Top-Down ARPG (가칭)

이 프로젝트는 자바(Java)와 Swing을 사용하여 개발 중인 간단한 2D 액션 RPG 게임입니다.

## ⚙️ 주요 기술 스택 및 환경

* **언어:** Java (JDK 17 이상 권장)
* **프레임워크/라이브러리:** Java Swing (GUI), AWT (Graphics)
* **개발 환경:** IntelliJ IDEA 또는 Eclipse

---

## 🛠️ 최근 주요 구현 사항 (몬스터 로직 및 히트박스 개선)

몬스터의 비주얼과 실제 충돌 영역(히트박스)이 일치하지 않아 발생하던 문제를 해결하고, 몬스터의 기본 동작 로직을 개선했습니다.

### 1. 몬스터 (`Enemy.java`) 클래스 개선

* **히트박스 통일:** 몬스터의 드로잉 크기 (`drawWidth`, `drawHeight`)와 충돌 판정 크기 (`hitWidth`, `hitHeight`)를 명확하게 분리했습니다.
* **자동 크기 설정:** 모든 몬스터는 이제 `loadImage()` 시에 **IDLE 애니메이션 스프라이트의 실제 픽셀 크기**를 히트박스와 드로잉 크기의 기본값으로 사용하도록 로직을 통일했습니다.
    * 이를 통해 몬스터별로 수동으로 크기를 설정하던 코드를 제거하여 일관성을 확보했습니다.
* **히트박스 시각화:** 디버깅을 위해 몬스터 스프라이트 주변에 **빨간색 사각형 (히트박스)**을 그리는 코드를 추가했습니다.
* **HP 바 위치 조정:** HP 바가 몬스터 이미지의 크기 변화에 관계없이 **히트박스 중앙 상단**에 위치하도록 계산 로직을 수정했습니다.

### 2. 몬스터 타입 (`EnemyType.java`) 설정

* **사거리 필드 추가:** 몬스터별 **공격 사거리**를 명시하는 `range` 필드를 추가했습니다.
    * `Enemy.java`에서 이 값을 기반으로 `attackRange`와 `moveRange`를 설정하여, 몬스터의 추적/공격 로직을 구현할 수 있는 기반을 마련했습니다.
* **몬스터 스탯 조정:** 슬라임, 늑대, 악어, 오크, 얼음 기사 등 **다수의 몬스터에 대해 사거리(Range)와 스탯(HP, 공격력, 속도)**을 게임 밸런스에 맞게 조정했습니다.

## 💻 현재 실행 상태

### 게임 화면 및 요소

* **몬스터:** 여러 종류의 몬스터가 생성되며, 디버깅을 위해 몬스터의 **이름**, **현재 HP**, **최대 HP**, 그리고 **히트박스 (빨간색 테두리)**가 화면에 표시됩니다.

### 클래스 구조

| 파일 | 역할 |
| :--- | :--- |
| `Main.java` | 게임 실행 진입점 (`MainFrame` 인스턴스 생성) |
| `MainFrame.java` | Swing 기반 메인 윈도우(JFrame) 설정 및 `GamePanel` 추가 |
| `Constants.java` | 게임 상수 (타일 크기, 화면 크기, 월드 크기 등) 정의 |
| `GamePanel.java` | 게임 루프(Thread, FPS), 입력(KeyListener), 렌더링(paintComponent) 관리 |
| `Enemy.java` | **몬스터 객체의 핵심 로직** (좌표, 스탯, 상태, 애니메이션 로딩, 드로잉, 히트박스) |
| `EnemyType.java` | **몬스터 종류별 기본 스탯 정의** (이름, HP, 속도, 공격력, 사거리) |
| `Boss.java` | (미구현) 보스 몬스터 관련 클래스 |

## 📂 `enemy` 모듈 수동 통합 가이드 (Manual Integration)

이 섹션은 Git 환경이 아닌 곳에서, 또는 별도로 관리되던 프로젝트에 **`enemy` 폴더**와 그 내부 파일들(`Enemy.java`, `EnemyType.java` 등)을 수동으로 통합하고자 할 때 필요한 절차를 안내합니다.


### 1\. 📝 필수 Import 및 의존성 설정

파일을 복사한 후, `game` 패키지의 파일들이 `enemy` 패키지를 인식하도록 `import` 문을 추가해야 합니다.

| 파일 | 필요한 수정 사항 | 설명 |
| :--- | :--- | :--- |
| **`GamePanel.java`** | `import enemy.Enemy;` | 몬스터 객체를 생성하고 업데이트하기 위해 필요합니다. |
| | `import enemy.EnemyType;` | 몬스터 타입 정보를 불러오기 위해 필요합니다. |
| **`Constants.java`** | **(변동 없음)** | 현재 `Constants.java`는 `enemy` 모듈에 의존하지 않습니다. |
| **`Enemy.java`** | **(변동 없음)** | `Enemy.java`는 외부 `Constants` 클래스를 직접 참조하지 않도록 설계되었습니다. |

### 2\. 🔩 `GamePanel.java` 코드 통합 예시

`GamePanel.java`의 `setupGame()` 함수 내에서 몬스터 객체를 생성하는 방식입니다. 이 방식은 `enemy` 모듈이 성공적으로 통합된 후에 작동합니다.

```java
// GamePanel.java - setupGame() 함수 내 예시

// 기본 몬스터 생성 (EnemyType과 좌표 사용)
enemies.add(new Enemy(EnemyType.SLIME, 100, 200));

// ★ 드로잉 크기를 수동으로 제어하며 객체를 생성하는 방법:
Enemy orc = new Enemy(EnemyType.ORC, 400, 200);
orc.drawWidth = 200;  // 몬스터의 너비를 200px로 설정
orc.drawHeight = 150; // 몬스터의 높이를 150px로 설정
enemies.add(orc);
```
